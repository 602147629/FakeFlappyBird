2015.02.12，新分支PoolRefactor  
现在的对象池存在的问题： 
不使用泛型，对象池功能过于耦合，包含对象池和对象池集合。之前设想可以被对象池管理的对象都实现Ipooled接口，现在参考了多方资料，觉得更合适的方案是使用组合把要被对象池管理的对象作为新的一个`对象池对象创建者`的引用。对象池再引用该对象。
